# حماية البيانات

آخر تحديث: 2026-07-04
الحالة: نشط

# مقدمة

يدير هذا النظام بيانات مرتبطة بمناسبات شخصية — مناسبات، دعوات، مدعوين، وحضوراً فعلياً لأشخاص حقيقيين. حماية هذه البيانات ليست تفصيلاً تقنياً ثانوياً، بل شرط جوهري لمصداقية المنتج بأكمله. يثق المضيف بأن تفاصيل مناسبته وقائمة مدعوّيه لن تُكشف أو تُستخدم خارج سياقها، ويثق المدعو بأن مشاركته في مناسبة معينة تبقى ضمن حدود تلك المناسبة فقط. هذه الثقة تحمي علاقات شخصية حقيقية (عائلية واجتماعية)، وتحافظ على سلامة العمليات التشغيلية للمنتج، من إنشاء المناسبة وحتى تسجيل الحضور.

# أنواع البيانات

## بيانات المناسبات

- تفاصيل المناسبة.
- الموقع.
- التاريخ.
- إعدادات المناسبة.

هذه البيانات تصف المناسبة كتعريف قائم بذاته، وتخص المضيف الذي أنشأها حصراً.

## بيانات المدعوين

- الهوية.
- معلومات التواصل.
- الردود.
- حالة الحضور.

هذه البيانات تخص الأشخاص المدعوّين لمناسبة محددة، وترتبط دائماً بتلك المناسبة تحديداً.

## بيانات المستخدمين

- الحساب.
- الباقة.
- التفضيلات التشغيلية.

هذه البيانات تصف المضيف كمستخدم للمنتج، بمعزل عن أي مناسبة بعينها.

## بيانات التكاملات

- معلومات الربط التجاري.
- إعدادات قنوات الاتصال.
- البيانات الضرورية لتشغيل الخدمات الخارجية.

هذه البيانات تدعم عمل التكاملات الخارجية فقط، ولا تمثّل جزءاً من الهوية الأساسية للمنتج أو بياناته الجوهرية.

# مبادئ حماية البيانات

- جمع أقل قدر ممكن من البيانات اللازمة لتحقيق الغرض التشغيلي فقط.
- استخدام كل بيانة لتحقيق غرض واضح ومحدد، لا لأغراض إضافية غير معلنة.
- عدم مشاركة البيانات بين المناسبات المختلفة، حتى لو كانت مملوكة لنفس المضيف.
- احترام استقلالية بيانات كل مستخدم عن بيانات المستخدمين الآخرين.
- عدم كشف البيانات الداخلية للمستخدمين أنفسهم أو لأي طرف خارجي دون ضرورة واضحة.

# استقلالية المناسبات

- كل مناسبة تمثّل مساحة بيانات مستقلة تماماً عن غيرها.
- المدعو يرى فقط ما يخص المناسبة التي تحقّقت هويته كمدعو لها.
- بيانات مناسبة واحدة لا تنتقل أو تُشارَك مع مناسبة أخرى، حتى لو تشابهت الأطراف المعنية.
- الصلاحيات الممنوحة لمناسبة معينة لا تنتقل تلقائياً لأي مناسبة أخرى.

# الاحتفاظ بالبيانات

- يُحتفظ بالبيانات الضرورية فقط لتشغيل الخدمة وتقديم قيمتها للمضيف والمدعو.
- تُتجنَّب حيازة أي بيانات لا تحمل قيمة تشغيلية واضحة.
- يُحترَم دورة حياة المناسبة؛ البيانات المرتبطة بمناسبة منتهية تُعامَل وفق طبيعة تلك المرحلة من حياتها.
- تُعامَل البيانات التاريخية كسجل تشغيلي يُرجَع إليه عند الحاجة، لا كوسيلة لتوسيع صلاحيات حالية أو إعادة فتح قرارات سابقة.

# العلاقة مع التكاملات الخارجية

- التكاملات الخارجية لا تمتلك البيانات الأساسية للمنتج بأي شكل من الأشكال.
- البيانات المرسلة للخارج تقتصر على الحد الأدنى اللازم لتنفيذ الخدمة المطلوبة فقط.
- فقدان مزود خارجي أو تعطّله لا يؤدي إلى فقدان أي بيانات داخلية قائمة.
- يحتفظ النظام دائماً بملكيته الكاملة للحالة التشغيلية للمستخدمين والمناسبات، بصرف النظر عن أي تكامل خارجي.

# ما لا يجب كسره

- خصوصية المدعوين لا تُضحّى أبداً من أجل الراحة التشغيلية أو سهولة التنفيذ.
- أقل قدر ممكن من البيانات يُستخدم دائماً في أي عملية أو ميزة.
- لا توجد مشاركة ضمنية للبيانات بين المناسبات المختلفة تحت أي ظرف.
- البيانات التشغيلية الأساسية تبقى داخل النظام، ولا تُدار من خارجه.
- التكاملات الخارجية ليست مصادر حقيقة للبيانات الأساسية للمنتج بأي حال.
