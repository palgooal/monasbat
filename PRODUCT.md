# Product

## Register

product

## Users

Hosts and event organizers using an Arabic RTL WordPress product to create events, manage invitees, send invitations, collect RSVP responses, and run check-in on the event day. Users are usually in a task flow after purchasing or activating a package, so clarity, quota visibility, and reliable form behavior matter more than marketing flourish.

## Product Purpose

Monasbat helps customers create and manage event invitation pages for weddings and events. Success means a host can create an event within their package limits, configure the details, manage guests, and trust that RSVP and check-in data will work without needing WordPress admin knowledge.

## Brand Personality

Clear, trustworthy, service-oriented. The interface should feel polished and calm, with enough warmth for social events but the discipline of an operational tool.

## Anti-references

Avoid over-decorated SaaS templates, excessive glassmorphism, generic gradient-heavy dashboards, unclear custom form controls, and marketing-style page sections inside authenticated task flows. Do not make users hunt through decorative content when they are trying to create or manage an event.

## Design Principles

1. Put the next operational action first.
2. Make package limits, blocked states, and required fields explicit before submission.
3. Preserve RTL clarity across every label, control, and validation message.
4. Use restrained visual polish that supports trust rather than distraction.
5. Keep WordPress implementation details invisible to non-technical hosts.

## Accessibility & Inclusion

Target WCAG AA for color contrast, keyboard navigation, visible focus, labeled form controls, and understandable validation. Support reduced motion and avoid color-only communication, especially for quota, feature availability, and form error states.
